Add a user onboarding survey to the account creation flow for this app.

Goal:
I want the survey to be part of sign-up / account creation so the app can personalize the user experience immediately. I also want all survey answers to be editable later inside the user’s profile/settings page. In addition, based on the user’s answers, I want the system to generate internal demographic / personalization tags that are stored in the database and used for tailored recommendations. These tags should not be directly shown to the user.

Please do all of the following:

Add an onboarding survey immediately after account creation, or as the final step of account creation if that fits the current auth flow better.

Save the user’s answers to their profile in Supabase.

Make the survey answers editable later in the profile/settings area.

Generate internal hidden personalization tags based on the user’s answers. These tags are for recommendation logic only and should not be displayed directly in the UI.

Keep the survey visually consistent with the site theme and current auth/profile styling.

Do not break the existing login, signup, logout, forgot password, or profile flows.

Use multi-step onboarding screens instead of one giant form.

Use chips / pills / multi-select buttons for interests and preference tags where appropriate.

Allow skipping certain optional questions if needed, but collect the most important ones.

Show progress through the onboarding flow.

Allow users to come back and edit this later in profile settings.
